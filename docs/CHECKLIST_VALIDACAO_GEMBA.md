# Checklist de Validação Gemba

Use este checklist antes de aceitar a planilha final ou transformar uma sugestão em mudança de processo.

## Identificação

- Posto:
- Processo:
- Responsável pela revisão:
- Data:
- Vídeo/fonte:
- Takt usado:

## Ciclo Observado

- [ ] O vídeo representa um ciclo real e típico da operação?
- [ ] Produto, variante, WO/KD/ROP e condição de abastecimento estavam corretos?
- [ ] Início do ciclo foi confirmado?
- [ ] Fim do ciclo foi confirmado?
- [ ] O ciclo observado no JSON/Excel corresponde à medição revisada?
- [ ] Há alerta quando o tempo total revisado diverge do ciclo observado?

## Microetapas

- [ ] As microetapas fazem sentido para quem conhece o posto?
- [ ] A sequência das ações está correta?
- [ ] Não há ações distintas agrupadas em uma única microetapa?
- [ ] Tempos de início, fim e duração foram revisados?
- [ ] Etapas de baixa confiança têm motivo registrado?
- [ ] A descrição usa nomenclatura conhecida e validada?

## AV/NAV/D

- [ ] AV foi validado como transformação direta ou requisito necessário do produto?
- [ ] NAV foi validado como necessário por segurança, qualidade, rastreabilidade, sistema ou padrão atual?
- [ ] D está realmente eliminável ou reduzível?
- [ ] Classificações alteradas manualmente foram justificadas?
- [ ] Nenhuma classificação responsabiliza operador sem evidência objetiva?

## Melhorias

- [ ] Toda etapa D tem melhoria, justificativa ou decisão registrada?
- [ ] As melhorias são seguras para teste?
- [ ] Ergonomia foi considerada?
- [ ] Qualidade foi considerada?
- [ ] Segurança foi considerada?
- [ ] Rastreabilidade e sistema foram considerados?
- [ ] Mudanças de método, layout, sequência, ferramenta ou abastecimento foram marcadas para validação gemba?

## Excel e Registros

- [ ] Planilha final foi revisada pelo responsável?
- [ ] JSON final foi salvo junto com o Excel?
- [ ] Resumo AV/NAV/D confere com as microetapas?
- [ ] Alertas de validação foram lidos?
- [ ] Template original foi preservado?
- [ ] Fórmulas e abas principais foram preservadas?

## Decisão

- [ ] Aprovado para teste controlado.
- [ ] Requer nova coleta de vídeo/dados.
- [ ] Requer ajuste manual da análise.
- [ ] Requer envolvimento de segurança/qualidade/ergonomia.
- [ ] Rejeitado para implementação.
